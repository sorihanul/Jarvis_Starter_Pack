# Coding Sequential Agent Starter (v1.2 Lite)

This module is a lightweight starter version of the common sequential execution series.

## Goal
Run coding tasks in fixed order to reduce drift:
1. Design
2. Validation
3. Implementation
4. Inspection

Optional phases:
5. Environment Alignment
6. Agenda Log

## Files
- `01_DESIGN_AGENT.md`
- `02_VALIDATION_AGENT.md`
- `03_IMPLEMENTATION_AGENT.md`
- `04_INSPECTION_AGENT.md`
- `05_ENV_ALIGNMENT_AGENT.md` (optional)
- `06_AGENDA_LOG_AGENT.md` (optional)
- `07_POSSESSION_ROUTER_v1.2.md`
- `08_PHASE_OUTPUT_TEMPLATES_v1.2.md`
- `09_REENTRY_CANON_v1.2.md`

## How to use (manual, no extra tooling)
1. Tell your agent to read this folder.
2. Tell it to execute phases in order.
3. Stop after each phase if you want human approval.
4. Use templates in `08_PHASE_OUTPUT_TEMPLATES_v1.2.md` for stable outputs.

## Recommended command sentence
"Read `01_Modules/Coding_Sequential_Agent_Starter` and run Design -> Validation -> Implementation -> Inspection in order."

## Note
This is starter-safe and intentionally minimal.
Use advanced runtime scripts from your main workspace only when needed.
