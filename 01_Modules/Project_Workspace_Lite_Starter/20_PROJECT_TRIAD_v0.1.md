# Project Triad v0.1

## 개요

프로젝트는 아래 3문서가 핵심이다.

## 1. MISSION

역할:
- 프로젝트 목적
- 해결할 문제
- 범위
- 성공 기준

질문:
- 왜 이 프로젝트를 하는가
- 이번에 어디까지 할 것인가
- 무엇이 되면 성공인가

## 2. ROADMAP

역할:
- 프로젝트 내부 단계
- 마일스톤
- 단기 진행 순서

질문:
- 어떤 순서로 전개되는가
- 지금은 어느 단계인가
- 다음 단계는 무엇인가

## 3. ORCHESTRATOR

역할:
- 실행 방식
- 역할 분담
- 핸드오프
- 검증과 종료 조건

질문:
- 누가 무엇을 하는가
- 순차형으로 갈지 검토형으로 갈지
- 언제 멈추고, 누가 승인하는가

## 관계

- `MISSION` = 왜
- `ROADMAP` = 어떻게 전개
- `ORCHESTRATOR` = 누가 어떻게 집행

작은 프로젝트는 이 셋으로 충분하다.
