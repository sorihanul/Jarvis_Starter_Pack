# Subagent Brief Template v0.1

아래 형식은 Codex 서브에이전트에게 일을 넘길 때 쓰는 최소 브리프다.

## 템플릿

```text
역할:
- research | implementation | validation | synthesis

목표:
- 이번 서브에이전트가 끝내야 할 한 가지 목표

범위:
- 읽을 파일/폴더
- 수정 가능한 파일/폴더

출력:
- 반드시 돌려줘야 할 결과물

금지:
- 하지 말아야 할 것

주의:
- 메인 세션이 통합자이며, 서브에이전트는 좁은 역할만 수행한다
```

## 예시 1: Research

```text
역할:
- research

목표:
- 현재 프로젝트 구조와 관련 문서를 읽고 핵심 진입 파일을 찾는다

범위:
- 읽기: TASKS/, 00_Core/, 01_Modules/
- 수정: 없음

출력:
- 핵심 파일 3~5개
- 읽기 순서
- 구조상 주의점 3개

금지:
- 구현 제안 확정
- 파일 수정
```

## 예시 2: Validation

```text
역할:
- validation

목표:
- 구현 결과가 출력 계약과 정책을 지키는지 점검한다

범위:
- 읽기: 변경된 파일, POLICY.md, 관련 TASKS/
- 수정: 없음

출력:
- 통과/실패
- 주요 위험 3개 이하
- 추가 확인 필요 항목

금지:
- 새 기능 구현
- 코어 레이어 변경
```
