# 1. 목적 (Purpose)
현재 요청의 범위를 짧고 강하게 잠가서, 세션이 불필요하게 팽창하지 않도록 합니다.

# 2. 입력 (Input)
- `현재 요청`
- (선택사항) `이미 열려 있는 세션 카드`
- (선택사항) `현재 프로젝트 또는 작업장 경로`

# 3. 절차 (Procedure)
1. 요청이 단발 작업인지, 프로젝트형 작업인지, 보안/승인이 먼저 필요한지 먼저 판단합니다.
2. 이번 턴에서 다룰 범위를 한 문장으로 제한합니다.
3. 이번 세션에서 다루지 않을 것을 1~3개로 적습니다.
4. 범위를 넘는 신호가 있으면 바로 `followup` 또는 별도 세션 필요 여부를 붙입니다.

# 4. 출력 (Output)
- **bounded_scope**: [이번 세션이 다룰 정확한 범위]
- **out_of_scope**: [이번 세션에서 다루지 않을 항목]
- **followup_if_needed**: [필요한 후속 세션 또는 다음 단계]

# 5. 실패 처리 (Fallback)
- **범위 모호**: 요청이 너무 넓으면 하나의 bounded scope로 줄이고, 나머지는 `followup_if_needed`로 넘깁니다.
