# 1. 목적 (Purpose)
특정 코드, 스크립트, 혹은 작성 문서를 검토하여, `POLICY.md`를 포함한 핵심 규칙 위반 사례 및 기술적 결함을 지적하고 개선안을 제시합니다.

# 2. 입력 (Input)
- `대상 파일 절대 경로`
- (선택사항) `특히 주의 깊게 봐야 할 요소` (예: 가독성, 메모리 누수, 맞춤법 등)

# 3. 절차 (Procedure)
1. 타겟 파일을 읽어 들인 후, 목적과 논리적인 흐름을 먼저 파악합니다.
2. 코드인 경우: 성능 병목, 보안 취약점, 하드코딩된 값, 에러 처리 미흡 여부를 검사합니다.
3. 문서인 경우: 문맥의 매끄러움, `한글 AILO-H` 교정 규칙, 톤(Tone) 위반 여부를 검사합니다.
4. 문제가 발견되는 즉시, 정확한 라인 번호와 문제 원인을 메모리에 기록합니다.
5. 발견된 결함들에 대한 더 나은 대체 방안(Refactored Code or Rewritten Text)을 생성합니다.

# 4. 출력 (Output)
- **리뷰 대상**: [파일명 또는 경로]
- **주요 발견점**: [위반 사항이나 버그에 대한 간단한 목록]
- **코드/문구 교정 제안**: 원래의 내용과 개선된 내용을 비교하는 스니펫 형식 (Diff 형식 권장)

# 5. 실패 처리 (Fallback)
- **파일 없음**: 타겟 파일 경로를 찾을 수 없거나 접근 권한이 없으면 "대상 파일 접근 불가. 경로를 확인해주세요."라고 응답하고 작업을 중지합니다.
