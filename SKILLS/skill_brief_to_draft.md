# 1. 목적 (Purpose)
짧은 브리프를 톤과 구조가 보이는 단일 초안으로 빠르게 전환합니다.

# 2. 입력 (Input)
- `브리프`
- `목적`
- (선택사항) `톤`
- (선택사항) `길이 제약`

# 3. 절차 (Procedure)
1. 브리프의 목표와 독자를 한 줄로 정리합니다.
2. 필요한 톤과 길이를 먼저 잠급니다.
3. 한 번에 읽히는 단일 초안을 작성합니다.
4. 군더더기 설명을 줄이고, 실제 사용 가능한 표면을 우선합니다.

# 4. 출력 (Output)
- **draft_goal**: [초안 목표]
- **tone_lock**: [톤]
- **draft**: [초안 본문]
- **revision_hint_if_needed**: [추가 수정 힌트]

# 5. 실패 처리 (Fallback)
- **브리프 부족**: 정보가 부족하면 억지로 늘리지 말고 가장 안전한 짧은 초안을 반환하고 부족한 점을 적습니다.
