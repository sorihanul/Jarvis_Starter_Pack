# 1. 목적 (Purpose)
코딩 결과를 장황한 설명이나 완성품 선언 대신, 작은 패치형 산출로 정리합니다.

# 2. 입력 (Input)
- `수정 목표`
- `대상 파일 또는 예상 파일 위치`
- (선택사항) `제약 조건`

# 3. 절차 (Procedure)
1. 무엇을 바꾸는지 한 줄로 적습니다.
2. 파일 위치를 명확히 적습니다.
3. 변경 의도를 1~3개 포인트로 줄입니다.
4. 필요한 경우 짧은 코드 조각이나 diff 성격의 제안을 붙입니다.
5. 결과를 `패치형 전달`로 유지하고, 완성품 과장을 피합니다.

# 4. 출력 (Output)
- **intended_file_location**: [대상 파일]
- **patch_intent**: [변경 의도]
- **implementation_slice**: [핵심 변경 조각]
- **placement_rationale**: [왜 그 위치인지]

# 5. 실패 처리 (Fallback)
- **파일 불명확**: 대상 파일이 불명확하면 임의 생성 대신 예상 위치와 이유만 반환합니다.
- **권한/환경 제한**: 실제 적용 불가면 패치형 제안까지만 반환합니다.
