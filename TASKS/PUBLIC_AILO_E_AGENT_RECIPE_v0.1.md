# Public AILO-E Agent Recipe v0.1

## Purpose

- define when and how to write a public `AILO-E` agent card inside Jarvis Starter
- keep stronger structure where it helps, without overloading every lane

## What Makes An AILO-E Public Card Different

- stronger scope honesty
- stronger route lock or stop posture
- stronger bounded return discipline
- more explicit escalation awareness

This is still a public card.
It should remain readable and usable inside Jarvis Starter without hidden context.

## Use AILO-E Only When

- route locking matters more than speed
- stop behavior matters more than output eagerness
- uncertainty separation matters more than smooth prose
- escalation or protected-boundary awareness matters

## Do Not Use AILO-E By Default When

- the lane is a small specialist lane
- patch-shaped output is the main goal
- quick public-generic output matters most
- the task is low-risk and tightly bounded

## Recommended Public AILO-E Lanes

- intake chief
- stronger research specialist
- blocker-honest coding worker

## Current Public Non-AILO Lanes

- coding specialist
- writing specialist
- lighter intake router

## Recommended Card Structure

1. `역할 (Role)`
- clearly state that this is an `AILO-E` public card
- state whether it is `chief`, `follower`, or another bounded lane

2. `범위 (Scope)`
- sharply define scope and non-scope

3. `입력 계약 (Input Contract)`
- keep inputs small and explicit

4. `출력 계약 (Output Contract)`
- bounded fields only
- prefer state plus next action over freeform essay

5. `주요 스킬 (Primary Skills)`
- pair with lock, evidence, patch, or gate skills as needed

6. `금지사항 (Forbidden)`
- forbid role drift
- forbid over-helpful filler

7. `종료 조건 (Exit Condition)`
- stop when the bounded contract is complete

## Public AILO-E Guardrails

- do not make a card heavy just because it sounds more advanced
- do not use AILO-E as prestige decoration
- do not overload specialist lanes with structure they do not need
- keep the card public-safe and self-contained

## Promotion Check

- the AILO-E lane has a tested reason to exist
- the same lane was compared against a lighter public form when relevant
- the stronger form actually improves discipline, honesty, or routing

## One-Line Verdict

- public AILO-E cards should exist only where stronger structure clearly beats lighter public forms
