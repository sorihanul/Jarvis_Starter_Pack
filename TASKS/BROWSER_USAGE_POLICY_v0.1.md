# Browser Usage Policy v0.1

## 목적
- 자비스 스타터에서 브라우저 사용이 무엇을 의미하는지 공개 사용자 기준으로 설명한다.
- 브라우저를 기본 전제로 강제하지 않고, 선택 가능한 보조 능력으로 정리한다.

## 한 줄 원칙
브라우저는 자비스 스타터의 기본 구성요소가 아니라, 환경이 허용할 때만 붙는 선택형 탐색/상호작용 능력이다.

## 기본 입장
1. 스타터는 설치 없이 문서만으로 시작 가능해야 한다.
2. 따라서 브라우저 사용 가능 여부는 환경 의존적이다.
3. 브라우저가 없어도 스타터의 핵심 흐름은 유지되어야 한다.

## 브라우저로 할 수 있는 일
- 웹 문서 읽기
- 공식 문서/페이지 확인
- 간단한 탐색과 구조 파악
- 필요 시 UI 상호작용 보조

## 브라우저 없이도 가능한 일
- 코어 문서 읽기
- 프로젝트 작업장 운영
- 세미 오토 프리셋 사용
- 학습 루프 운영
- 로컬 IVK 기반 회수
- 문서형 오케스트레이션 설계

## 권장 우선순위
1. 먼저 로컬 문서와 IVK를 본다.
2. 로컬 근거가 부족할 때만 웹 탐색을 고려한다.
3. 브라우저 자동화는 정말 필요할 때만 사용한다.

## 공개판 설명 원칙
- 브라우저는 `필수 기능`처럼 소개하지 않는다.
- `사용 가능한 환경에서는 활용 가능` 수준으로 설명한다.
- 브라우저 사용 불가 환경에서도 스타터가 충분히 쓸 수 있음을 분명히 한다.

## 기대치 조정
### 기대 가능한 것
- 외부 자료 확인 보조
- 웹 기반 리서치 보조
- 브라우저가 허용된 환경에서의 탐색

### 기대하면 안 되는 것
- 모든 환경에서 동일한 브라우저 동작 보장
- 브라우저 자동화가 기본적으로 포함된다고 가정
- 브라우저가 없으면 스타터를 쓸 수 없다는 해석

## 메인 세션에서의 위치
- 메인 세션은 브라우저를 필수 전제로 두지 않는다.
- 리서치 또는 검증이 필요할 때만 선택적으로 호출한다.

## 프로젝트 오케스트레이션에서의 위치
- 프로젝트 요구사항에 따라 필요 시 브라우저 사용 가능 여부를 명시한다.
- 브라우저 사용이 핵심이면 프로젝트 문서에 따로 적는다.
- 브라우저가 없어도 진행 가능한 대체 루프를 함께 적는 것을 권장한다.

## 공개판 권장 문구
- "브라우저 사용은 선택 사항이며, 환경이 허용할 때만 활용합니다."
- "스타터의 핵심 흐름은 브라우저 없이도 동작하도록 설계되어 있습니다."
- "웹 탐색은 로컬 문서와 IVK가 부족할 때의 보조 수단입니다."
