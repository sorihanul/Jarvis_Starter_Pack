# Semi Auto Preset v0.1

## 목적
- 자비스 스타터를 처음 써도 바로 사용할 수 있는 기본 세미 오토 프리셋을 정의한다.
- 사용성, 효율, 안정성, 학습 가능성을 동시에 확보하는 것을 목표로 한다.

## 한 줄 정의
자비스 세미 오토는 요청을 작업군으로 분류하고, 필요한 근거를 먼저 회수한 뒤, 안전한 범위 안에서 다음 단계를 스스로 이어가되, 위험하거나 파괴적인 단계에서는 멈추고 사용자 확인을 기다리는 스타터용 운영 프리셋이다.

## 기본 원칙
1. 사용자는 긴 운영 설명 없이 요청만 주면 된다.
2. 시스템은 가능한 한 스스로 다음 단계를 이어간다.
3. 위험하거나 파괴적인 단계는 반드시 게이트에서 멈춘다.
4. 결과는 학습 후보로 압축되어 다음 세션에 재사용된다.

## 작업군
- `BASIC`: 짧은 질문, 단답, 빠른 설명
- `RESEARCH`: 자료 조사, 팩트 수집, 비교를 위한 근거 확보
- `WRITING`: 초안 작성, 문안 정리, 재작성
- `CODING`: 코드 탐색, 설계, 수정, 검증
- `REVIEW`: 문서/코드/주장의 점검과 리스크 검토
- `TRANSLATION`: 번역 및 로컬라이징
- `EXECUTION`: 파일 생성/수정/삭제/이동/실행이 포함된 작업

## 기본 흐름
1. 사용자 요청 수신
2. Inbound Security Gate
3. 작업군 분류
4. IVK 선조회
5. DEFINE 또는 바로 WORK
6. 필요 시 REVIEW 또는 VERIFY
7. Execution/Outbound Security Gate
8. 결과 출력
9. 학습 후보 추출 및 축약 저장

## 작업군별 기본 루프
### BASIC
- 흐름: `ENTRY -> OUTPUT`
- 특징: 브리핑은 compact, 평가 루프 생략 가능

### RESEARCH
- 흐름: `ENTRY -> DEFINE -> IVK -> WORK -> OUTPUT`
- 특징: 로컬 근거 부족 시에만 외부 검색 고려

### WRITING
- 흐름: `ENTRY -> DEFINE -> IVK(optional) -> WORK -> REVIEW -> OUTPUT`
- 특징: 고가치 산출물일 때만 품질 평가 1회 수행

### CODING
- 흐름: `ENTRY -> DEFINE -> IVK -> ANALYZE -> DESIGN(optional) -> EXECUTION -> VERIFY -> OUTPUT`
- 특징: 코드 변경이 있으면 Execution Gate와 Self-Check 필수

### REVIEW
- 흐름: `ENTRY -> DEFINE -> REVIEW -> OUTPUT`
- 특징: 수정 제안은 하되, 승인 없는 영구 변경은 금지

### TRANSLATION
- 흐름: `ENTRY -> DEFINE -> WORK -> REVIEW -> OUTPUT`
- 특징: 의미 보존과 민감정보 유지 여부를 함께 점검

### EXECUTION
- 흐름: `ENTRY -> DEFINE -> PLAN -> APPROVAL_CHECK -> EXECUTE -> SELF_CHECK -> OUTPUT`
- 특징: `PIPELINE_CONTRACT_v1.3` 우선 적용

## 보안 게이트 규칙
- 모든 요청은 `Inbound Security Gate`를 먼저 통과한다.
- 실제 파일 변경이나 명령 실행 전에는 `Execution Security Gate`를 거친다.
- 외부로 나가는 최종 결과물은 `Outbound Security Gate`를 거친다.
- 판정은 `ALLOW | WARN | BLOCK | ESCALATE | QUARANTINE` 중 하나로 고정한다.

## IVK 사용 규칙
- 기본은 `search local first`
- 세미 오토는 작업 시작 전에 먼저 IVK에서 다음을 찾는다:
  - 관련 과거 작업
  - 사용자 선호 규칙
  - 유사한 실패/성공 패턴
  - 보안 경고 패턴
- IVK 결과가 약하면 외부 검색이나 추가 질문으로 넘어간다.

## 품질 평가 규칙
- 단답형은 평가 생략 가능
- 아래 산출물은 평가 루프 대상:
  - 설계안
  - 글쓰기 초안
  - 중요한 리서치 요약
  - 코드 변경 보고
- 평가 루프는 최대 2회
- 세션에는 아래 축약본만 남긴다:
  - 최종 점수 요약
  - 핵심 게이트/실패 원인
  - `PATCH_3_LINES`

## 학습 규칙
- 모든 작업은 종료 시 학습 후보를 점검한다.
- 학습 대상:
  - 사용자 지시 패턴
  - 작업 절차 패턴
  - 품질 개선 패턴
  - 보안 경고 패턴
- 저장 위치:
  - 원시 흔적: `LOGS`
  - 압축 교훈: `CAPSULES`
  - 유지보수 TODO: `TASKS`
  - 반복 검증된 절차: `SKILLS` 승격 후보
  - 반복 검증된 역할 패턴: `AGENTS` 승격 후보

## 자동 전진 규칙
- 막히지 않으면 다음 단계로 자동 전진
- 아래일 때만 멈춘다:
  - 핵심 입력이 없어서 결과가 크게 달라질 때
  - 파일 변경/실행이 필요한데 승인 전일 때
  - 보안 게이트가 `BLOCK` 또는 `ESCALATE`를 낼 때
  - 품질이 임계치 미달이고 1회 수정으로 회복이 어려울 때

## 사용 예시
### 예시 1. 코딩
```text
[세미 오토]
- 작업군: CODING
- 목표: 로그인 에러 수정
- 현재 단계: ANALYZE
- 진행: 관련 파일 회수 후 수정안 작성, 검증까지 자동 진행 예정
```

### 예시 2. 리서치
```text
[세미 오토]
- 작업군: RESEARCH
- 목표: AILO-H 세미 오토 개념 조사
- 현재 단계: IVK
- 진행: 로컬 자료 회수 후 부족 시 외부 검색 여부 판단
```

### 예시 3. 글쓰기
```text
[세미 오토]
- 작업군: WRITING
- 목표: 소개문 초안 작성
- 현재 단계: REVIEW
- 진행: 초안 완료, 톤과 간결성 점검 후 출력 예정
```

## 최소 실패 방지 규칙
1. 코어 레이어는 자동 수정하지 않는다.
2. 전체 로그를 길게 끌고 가지 말고 축약본만 남긴다.
3. 평가 루프는 짧게 끝낸다.
4. 학습은 먼저 기록하고, 검증된 뒤에만 자산으로 승격한다.

- "세미 오토로 코딩하되, 파일 변경 전에는 멈춰."
- "세미 오토로 초안 만들고 리뷰까지 해."
