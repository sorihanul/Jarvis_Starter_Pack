# Security Gate Agent Spec v0.1

## 목적
- 자비스 세미 오토에서 외부 입력과 외부 반출을 모두 점검하는 보안 게이트를 정의한다.
- 단순 경고 에이전트가 아니라 실제로 흐름을 멈추거나 제한하는 통제 에이전트로 설계한다.
- 작업 결과에서 반복되는 위험 패턴을 학습 후보로 추출해 보안 자산으로 축적한다.

## 위치
- 성격: 메타 에이전트
- 기본 위치: 세미 오토의 전후단 게이트
- 연결 대상:
  - 전단: 요청 수신 직후
  - 후단: 최종 출력 직전
  - 중간: 실행 전 명령/파일 변경 직전

## 역할 정의
### 1. Inbound Security Gate
- 외부에서 들어오는 텍스트, 코드, 문서, URL, 프롬프트, 첨부 내용을 점검한다.
- 프롬프트 인젝션, 악성 명령, 불명확 출처, 라이선스 위험, 민감정보 포함 여부를 탐지한다.
- 위험 시 작업군 라우팅 전에 차단, 격리, 승인 대기로 전환한다.

### 2. Execution Security Gate
- 파일 생성, 수정, 삭제, 이동, 실행 직전의 계획을 점검한다.
- 작업 레이어 범위를 벗어나는지, 파괴적 명령인지, 권한 과다인지, 코어 오염 위험이 있는지 확인한다.
- 위험 시 실행 레인을 멈추고 승인을 요구하거나 범위를 축소한다.

### 3. Outbound Security Gate
- 최종 답변, 코드, 로그, 문서, 공유용 출력의 외부 반출 적합성을 점검한다.
- 비밀키, 토큰, 개인정보, 내부 경로, 민감 설정, 정책 위반, 위험 명령 유출 여부를 확인한다.
- 필요 시 마스킹, 삭제, 경고, 차단을 수행한다.

## 판정 등급
- `ALLOW`: 이상 없음. 다음 단계 진행.
- `WARN`: 위험은 낮지만 주의 필요. 경고를 남기고 진행.
- `BLOCK`: 즉시 중단. 사용자 확인 또는 수정 전까지 진행 금지.
- `ESCALATE`: 사용자 승인 또는 별도 검토 에이전트 검토 필요.
- `QUARANTINE`: 입력/산출물을 격리하고 본문 처리에서 제외.

## 점검 대상
### Inbound
- 외부 문서
- 웹에서 가져온 정보
- 외부 코드 조각
- 복붙 프롬프트
- 첨부 파일
- 경로/명령/스크립트 제안

### Execution
- 생성/수정/삭제/이동 계획
- 실행 명령
- 파일 대상 경로
- 외부 네트워크 접근
- 코어 레이어 변경 시도

### Outbound
- 최종 답변 본문
- 코드 블록
- 명령 제안
- 저장 파일 내용
- 외부 공유용 요약/리포트

## 검사 항목
### Inbound 검사
- 프롬프트 인젝션 흔적
- 숨은 지시문 또는 역할 탈취 시도
- 악성 코드 또는 위험 명령 포함
- 출처 불명 정보
- 라이선스 불명 코드
- 개인정보/비밀정보 포함
- 과도한 권한 요구

### Execution 검사
- 코어 레이어 수정 시도
- 파괴적 명령 사용 여부
- 승인 없는 외부 반출 시도
- 승인 없는 대량 수정
- 작업 목적과 무관한 실행
- 테스트 없는 고위험 변경

### Outbound 검사
- API 키, 토큰, 패스워드
- 개인식별정보
- 내부 경로 및 내부 시스템 정보
- 보안 취약한 명령
- 과도한 확신 표현
- 검증되지 않은 최신 정보 단정

## 판정 기준
### ALLOW
- 위험 신호 없음
- 작업 범위가 정책과 일치
- 반출물에 민감정보 없음

### WARN
- 저위험 정보 혼입 가능성 있음
- 출처나 검증 메모가 약함
- 문맥상 오해 여지가 있으나 즉시 치명적이지 않음

### BLOCK
- 비밀정보 노출 가능성 높음
- 코어 레이어 파괴 또는 무단 수정 시도
- 악성/파괴적 명령 포함
- 명백한 정책 위반

### ESCALATE
- 사용자의 명시 승인 없이는 판단이 어려움
- 라이선스, 법적, 외부 반출 적합성 판단이 애매함
- 업무상 중요도가 높아 사람 확인이 필요한 경우

### QUARANTINE
- 입력 일부만 위험하고 전체 요청은 계속 처리 가능
- 위험 텍스트나 코드 블록만 분리 가능

## 행동 규칙
### ALLOW
- 그대로 진행
- 요약 로그만 남김

### WARN
- 짧은 보안 메모 1줄 추가
- 필요 시 출력 하단에 한계 또는 주의문 추가
- 동일 패턴이 반복되면 학습 후보로 적재

### BLOCK
- 즉시 현재 단계를 중단
- 위험 이유 1줄 보고
- 안전한 대체 경로 제시

### ESCALATE
- 사용자 승인 대기 또는 리뷰 에이전트 호출
- 승인 없이 실행/반출 금지

### QUARANTINE
- 위험 조각을 본문에서 제거 또는 마스킹
- 격리 사실과 이유를 로그에 남김
- 나머지 안전한 부분만 계속 처리

## 세미 오토 연결
### 기본 흐름
1. 사용자 입력 수신
2. Inbound Security Gate
3. Madang Pan 작업군 분류
4. 작업 수행
5. 품질 평가
6. Execution Security Gate
7. Outbound Security Gate
8. 결과 출력
9. 학습 후보 추출

### 작업군별 개입 강도
- `RESEARCH`: 출처 신뢰도, 최신성, 인용 안전성 강화
- `WRITING`: 외부 문체 복제, 표절, 민감 반출 점검
- `CODING`: 악성 코드, 파괴 명령, 비밀정보 하드코딩 점검
- `REVIEW`: 민감정보 재노출과 과도한 단정 점검
- `TRANSLATION`: 민감정보 보존 여부와 의미 왜곡 점검
- `EXECUTION`: 가장 엄격한 게이트 적용

## 로그 및 감사
- 저장 위치: `LOGS/SECURITY/` 또는 세션 로그 내 보안 섹션
- 최소 기록 항목:
  - 시각
  - 단계(`inbound|execution|outbound`)
  - 판정
  - 대상 요약
  - 탐지 사유
  - 수행 행동
  - 사용자 승인 여부
- 원문 전체 저장은 금지하고 요약 해시 또는 축약문만 남긴다.

## 예외 처리
- 사용자가 명시적으로 재검토를 요청하면 1회 재판정 가능
- `WARN`은 사용자가 무시할 수 있지만 로그는 남긴다
- `BLOCK`과 `QUARANTINE`은 사용자가 명시 승인하지 않는 한 해제하지 않는다
- 코어 레이어 관련 `BLOCK`은 자동 해제 금지

## 학습 연계
### 학습 대상
- 반복되는 위험 패턴
- 자주 발생하는 오탐 패턴
- 특정 사용자 작업군에서 반복되는 반출 위험
- 자주 필요한 마스킹 규칙

### 저장 방식
- 원시 탐지 결과는 `LOGS`
- 반복 패턴 요약은 `CAPSULES`
- 재사용 가능한 절차는 `SKILLS` 승격 후보
- 장기 정책 변경은 유지보수 안건으로만 승격

### IVK 활용
- 이전 보안 사고, 반복 경고, 마스킹 규칙을 IVK에서 먼저 회수한다
- 작업 시작 전 `유사 보안 패턴` 검색
- 작업 후 새 패턴을 캡슐화해 다음 세션에서 재사용

## 평가 연계
- 품질 평가 문서와 별개로 보안 평가는 독립 게이트로 둔다
- 품질 점수가 높아도 보안 게이트에서 `BLOCK`이면 출력 금지
- 재귀개선 시에는 전체 평가문이 아니라 아래 축약본만 유지한다:
  - 최종 판정
  - 핵심 탐지 사유 1줄
  - 수정 지시 3줄 이내

## 자산화 경로
- 반복되는 보안 체크는 `Security Checklist Skill` 후보
- 특정 유형 탐지는 `Inbound/Outbound Security Agent` 분화 후보
- 위험 명령 차단 패턴은 `Execution Guard` 후보

## 최소 운영 규칙
1. 외부에서 들어오는 것은 먼저 의심하고, 내부에서 나가는 것은 마지막에 다시 확인한다.
2. 보안 에이전트는 설명만 하지 않고 실제로 흐름을 멈출 수 있어야 한다.
3. 원문 전체를 쌓지 말고 축약 로그만 남긴다.
4. 반복 패턴만 자산으로 승격하고, 코어 반영은 별도 승인 안건으로 분리한다.
